Application architecture

1. Goals

Our web solution provides a platform for different poems to be translated, commented and annotated. The site will provide a chart containing information about the activites done by the users. It will also have a feed where users and guests can see latest and/or most relevant modifications or additions to thesite. The latest translations will be avalible via a RSSFeed.

2. Database

The database consists of 7 tables which atomical data.

3. MVC-pattern

Our app is based on the MVC-pattern. Each page will contain 3 components:

  • View: used to show the representation of the content to the user.
  • Model: used to constantly make changes between the database and the view. Without the Model, the application would lose the avalibility of the contents from the controller.
  • Controller: The MySQL database that commits all the changes and allows the Model to easily access previously added info and content.

4. Application Scenarios

  1. A user opens the application.- The first page displayed will be the feed that displays the latest annotations and translations added by users, a small chart of users and the search bar.- From the feed he can log in, look for a poem, add a poem or just navigate through the page to see the news.
  2. The user wants to register in the application:-Uses the button "CREATE ACCOUNT"-This is redirected to the new account page, where he will enter his username, password, and email address.-If the email address is already used for another account or the password does not meet the requirements (minimum 8 characters alphanumeric with a minimum of one digit + one small letter + one big letter), an error message will be displayed, after which the user will be allowed to reintroduce the correct data.-If the entered data is correct, the user will be redirected to the account page from where he can manage his data.
  3. Administration of account data-The user enters the account administration button at the top right of any page-This button replaces the "CREATE ACCOUNT" and "LOG IN"-He may change his username, password , see the comments he added and delete or go to the page withthat poem4. Log in-It will be made through the "LOG IN" button in the upper right corner.-Redirected to the login page, the user can enter the name and password, along with the "remember me" option.-If the entered data does not match any account, the user will be able to re-enter the data.
  4. Access a poem:-The user can search for a poem by name or text from the search bar, once found the poem will be ableto enter the page of his poem and his translations by clicking on them-If the search poem is in the feed, a click will redirect the user to the requested page.-If the search poem is not found, the search page will display "No results found"
  5. Addition of poems:-From the administration page, the user can click the "ADD POEM" button from the main bar. It will be redirected to the poem adding page where he will fill the title, author, text fields.
  6. Addition of translations:-From the page of a poem, the user can click the "ADD TRANSLATION" button on the sidebar, he will be redirected to a page where he can adapt the language of the translation and the text.
  7. Adding annotations to the text:-From the page of a translated poem or original poem, the user will be able to select the text he wants to explain and will be redirected to a page where he can add annotation text or modify an existing annotation. Existing annotations can be changed, up-voted or down-voted.
  8. Adding comments:From the page of a translated poem or poem, the user can use the comments section at the bottom of the page to tell their opinions about text or translation.
  9. Follow the authors:From the poem or translation page added by a particular author, the user has the option to follow that author through the "FOLLOW" button, thus receiving notifications when he posts.
